# Local Catalog Build

This keeps hosted Supabase lean: build catalog data locally, then sync only
`public.products` to hosted.

## 1. Start local Postgres

```powershell
docker run --name lekkerlijst-catalog-postgres `
  -e POSTGRES_PASSWORD=localdev `
  -p 54322:5432 `
  -d postgres:16

$env:LOCAL_CATALOG_DB_URL = "postgresql://postgres:localdev@localhost:54322/postgres"
```

## 2. Bootstrap schema

```powershell
psql $env:LOCAL_CATALOG_DB_URL -f scrapers/local_catalog_schema.sql
```

The bootstrap creates `pgcrypto`, `pg_trgm`, `catalog.bronze_products`,
`catalog.silver_products`, `catalog.name_canonical`, local `public.products`,
and the build/search functions.

## 3. Seed paid work from hosted

Do not dump bronze. It is raw staging and can be regenerated by scraping.

```powershell
$env:SUPABASE_DB_URL = "<hosted pooler connection string>"

pg_dump $env:SUPABASE_DB_URL `
  --data-only `
  --table=catalog.silver_products `
  --table=catalog.name_canonical `
  --file=Output/catalog_seed.sql

psql $env:LOCAL_CATALOG_DB_URL -f Output/catalog_seed.sql
```

## 4. Rebuild local app products

```powershell
psql $env:LOCAL_CATALOG_DB_URL -c "select * from catalog.promote_to_public(null);"
psql $env:LOCAL_CATALOG_DB_URL -c "select catalog.apply_canonical_keys();"
psql $env:LOCAL_CATALOG_DB_URL -c "select * from public.search_product_offers('komkommer');"
```

## 5. Optional grouping cleanup

Free deterministic cleanup:

```powershell
psql $env:LOCAL_CATALOG_DB_URL -c "select * from catalog.apply_canonical_prepass();"
psql $env:LOCAL_CATALOG_DB_URL -c "select catalog.apply_canonical_keys();"
```

Targeted AI cleanup:

```powershell
$env:OPENAI_API_KEY = "<openai key>"
$env:OPENAI_CATALOG_MODEL = "gpt-5.4-mini"
$env:SUPABASE_DB_URL = $env:LOCAL_CATALOG_DB_URL

.venv\Scripts\python.exe -m scrapers.canonical_tag recanon-build --limit 500
.venv\Scripts\python.exe -m scrapers.canonical_tag submit-chunks
.venv\Scripts\python.exe -m scrapers.canonical_tag wait-chunks --interval 60
```

Remove `--limit` for the full targeted recanon pass after the pilot looks sane.

## 6. Sync hosted products

Apply `supabase/migrations/202606270002_prepare_catalog_decouple.sql` first.
Then:

```powershell
$env:LOCAL_CATALOG_DB_URL = "postgresql://postgres:localdev@localhost:54322/postgres"
$env:SUPABASE_DB_URL = "<hosted pooler connection string>"

.venv\Scripts\python.exe -m scrapers.sync_products --dry-run
.venv\Scripts\python.exe -m scrapers.sync_products
```

After sync, verify hosted:

```sql
select count(*) from public.products;
select * from public.search_product_offers('komkommer');
```

Only after those checks pass should hosted `catalog` be dropped.
